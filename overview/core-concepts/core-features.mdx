## Main feature areas

<Tabs>
  <Tab title="Analytics">
    ## Analytics

    Analytics helps you understand how users interact with your app or website.

    | Feature | Basic meaning |
    | --- | --- |
    | Users | People who visit your website or use your app |
    | Events | Actions users perform, like click, page view, add to cart |
    | Sessions | A period of user activity |
    | Screens & Pages | Pages or app screens users visit |
    | Reports | Visual view of user activity and performance |

    <Tip>
      Use Analytics when you want to understand user behavior and product usage.
    </Tip>
  </Tab>

  <Tab title="Ads Analytics">
    ## Ads Analytics

    Ads Analytics helps you understand how ads perform and how users behave after ad interaction.

    | Feature | Basic meaning |
    | --- | --- |
    | Ad Impressions | How many times ads were shown |
    | Ad Clicks | How many users clicked ads |
    | Ad Revenue | Revenue generated from ads |
    | Ad Failures | Ads that failed to load or display |
    | Campaign Performance | How paid campaigns bring users and conversions |

    <Tip>
      Use Ads Analytics when you want to measure ad performance, revenue, and campaign impact.
    </Tip>
  </Tab>

  <Tab title="Notifications">
    ## Notifications

    Notifications help you engage users and bring them back to your app or website.

    | Feature | Basic meaning |
    | --- | --- |
    | Create Notification | Build a notification campaign |
    | Target Users | Send notifications to selected users or segments |
    | Schedule Notification | Send notification immediately or later |
    | Notification Reports | Track sent, delivered, opened, clicked, and dismissed |
    | Engagement | Understand how users respond to notifications |

    <Tip>
      Use Notifications when you want to improve user engagement and retention.
    </Tip>
  </Tab>

  <Tab title="Integrations">
    ## Integrations

    Integrations help Twinalyze connect with external platforms.

    | Integration type | Examples |
    | --- | --- |
    | Ads | Google Ads, Facebook Ads, AdMob, AppLovin |
    | Tag Management | Google Tag Manager |
    | App Analytics | Firebase |
    | Notifications | OneSignal |
    | Team Alerts | Slack, Microsoft Teams |

    <Tip>
      Use Integrations when you want to connect Twinalyze with your existing marketing, ads, or communication tools.
    </Tip>
  </Tab>
</Tabs>

## How these features work together

<Steps>
  <Step title="Collect data">
    Twinalyze collects users, events, sessions, ads, and notification activity from your app or website.
  </Step>

  <Step title="Organize data">
    The data is connected with users, events, properties, sources, and campaigns.
  </Step>

  <Step title="Analyze performance">
    You can view reports for analytics, ads, notifications, and user behavior.
  </Step>

  <Step title="Improve growth">
    Use insights to improve conversion, retention, engagement, and ROI.
  </Step>
</Steps>

## Simple example

A user visits your website, clicks a product, adds it to cart, and later receives a notification.

| Twinalyze feature | What it tracks |
| --- | --- |
| Analytics | Page view, click, add to cart, session, scroll |
| Ads Analytics | Campaign source, ad click, conversion |
| Notifications | Notification sent, opened, clicked |
| Integrations | Google Ads, OneSignal, Firebase, GTM |
