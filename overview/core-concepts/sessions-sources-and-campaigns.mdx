Twinalyze doesn’t just track events — it helps you understand **when users come, where they come from, and how they behave during a visit**.

> **Session = when user is active**  
> **Source = where user came from**  
> **Campaign = why user came**

---

## Sessions

A **session** is a single visit or usage period.

It starts when a user opens your app or website and ends after inactivity or exit.

<CardGroup cols={2}>
  <Card title="Session start" icon="play">
    Triggered when user opens app or website.
  </Card>

  <Card title="Session end" icon="stop">
    Ends after inactivity or when user leaves.
  </Card>
</CardGroup>

### Example

User opens app → browses products → adds to cart → leaves  

➡️ All actions belong to **one session**

---

## Sources

A **source** tells you where the user came from.

This helps you understand traffic channels.

<AccordionGroup>
  <Accordion title="Organic">
    User came directly or through search without ads.
  </Accordion>

  <Accordion title="Ads">
    User came through paid campaigns (Google Ads, Facebook Ads).
  </Accordion>

  <Accordion title="Referral">
    User came from another website or app.
  </Accordion>

  <Accordion title="Direct">
    User directly opened your app or typed your website.
  </Accordion>
</AccordionGroup>

---

## Campaigns

A **campaign** helps you track marketing performance.

It usually comes from UTM parameters or integrations.

<CardGroup cols={2}>
  <Card title="UTM Campaigns" icon="tag">
    Track campaigns using UTM parameters like source, medium, campaign.
  </Card>

  <Card title="Ad Campaigns" icon="bullhorn">
    Track paid campaigns from Google Ads, Facebook Ads, and others.
  </Card>
</CardGroup>

---

## How they work together

<Steps>
  <Step title="User arrives">
    Twinalyze detects the source (ads, organic, referral, direct).
  </Step>

  <Step title="Session starts">
    A new session begins when the user enters your product.
  </Step>

  <Step title="User performs actions">
    Events like page views, clicks, and purchases are tracked.
  </Step>

  <Step title="Campaign is attached">
    If available, campaign data is linked to the session.
  </Step>
</Steps>

---

## Example journey

```txt
User clicks Facebook Ad
  ↓
Source = Ads
Campaign = Summer Sale
  ↓
Session starts
  ↓
page_view → product_view → add_to_cart → purchase


```
