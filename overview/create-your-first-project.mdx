# Create Project

A project is the main workspace where Twinalyze collects analytics data for your app or website.

You can create separate projects for **Web**, **Android**, and **iOS** so each platform has clean data, separate SDK credentials, and accurate reporting.

<CardGroup cols={3}>
  <Card title="Web Project" icon="globe">
    Create a project for websites, web apps, JavaScript SDK, React, Shopify, WordPress, or Webflow.
  </Card>

  <Card title="Android Project" icon="smartphone">
    Create a project for Android apps using package name, app name, and platform details.
  </Card>

  <Card title="iOS Project" icon="apple">
    Create a project for iOS apps using bundle ID, app name, and platform details.
  </Card>
</CardGroup>

## Before you begin

Make sure you have already completed these steps:

- Created your Twinalyze account
- Created your organization
- Logged in to the Twinalyze dashboard
- Selected the organization where you want to create the project

<Info>
  Create a separate project for every platform. For example, if you have one website and one Android app, create two different projects: one for Web and one for Android.
</Info>

---

## Project creation flow

<Steps>
  <Step title="Open your Twinalyze dashboard">
    Log in to your Twinalyze account and open the dashboard.
  </Step>

  <Step title="Select your organization">
    Choose the organization where you want to create the project.
  </Step>

  <Step title="Click create project">
    Open the project creation screen from your dashboard or organization area.
  </Step>

  <Step title="Choose project platform">
    Select Web, Android, or iOS based on your product.
  </Step>

  <Step title="Add project details">
    Fill in platform-specific details like website URL, package name, or bundle ID.
  </Step>

  <Step title="Save project">
    Create the project and continue to API keys or SDK setup.
  </Step>
</Steps>

## Choose your platform

Use the platform tabs below to understand which details you should enter for each project type.

<Tabs>
  <Tab title="Web">
    ## Create Web Project

    Choose **Web** if you want to track a website, web app, React app, Shopify store, WordPress site, or Webflow site.

    <CardGroup cols={2}>
      <Card title="Best for" icon="globe">
        Websites, landing pages, ecommerce stores, web apps, and browser-based products.
      </Card>

      <Card title="SDK type" icon="code">
        JavaScript SDK, React SDK, NPM package, Shopify, WordPress, or Webflow integration.
      </Card>
    </CardGroup>

    ### Required details

    | Field | Example | Description |
    | --- | --- | --- |
    | Project Name | `Twinsho Web` | Name of your website or web app project. |
    | Platform | `Web` | Select Web as the project platform. |
    | Website URL | `https://twinsho.com` | Main domain where SDK will be installed. |
    | Time Zone | `Asia/Kolkata` | Used for reports and date-based analytics. |
    | Environment | `Production` | Use production for live website tracking. |

    ### Recommended Web setup

    ```json
    {
      "projectName": "Twinsho Web",
      "platform": "Web",
      "websiteUrl": "https://twinsho.com",
      "timezone": "Asia/Kolkata",
      "environment": "Production"
    }
    ```

    <Warning>
      Make sure the website URL is correct. If your website has both `www` and non-`www` versions, use the main version that your users visit.
    </Warning>

## Project fields explained

| Field | Required | Used For |
| --- | --- | --- |
| Project Name | Yes | Displaying project in dashboard and reports. |
| Platform | Yes | Selecting SDK type and data structure. |
| Website URL | Web only | Identifying website traffic and allowed domain. |
| Package Name | Android only | Identifying Android app during SDK initialization. |
| Bundle ID | iOS only | Identifying iOS app during SDK initialization. |
| Time Zone | Recommended | Correct date and time in analytics reports. |
| Environment | Recommended | Separating production, staging, or testing data. |

## Best practices

<AccordionGroup>
  <Accordion title="Create separate projects for Web, Android, and iOS">
    Separate projects keep your analytics clean. It becomes easier to compare platform performance, debug SDK issues, and manage credentials.
  </Accordion>

  <Accordion title="Use clear project names">
    Use names like `Brand Web`, `Brand Android`, and `Brand iOS` instead of generic names like `Test` or `Project 1`.
  </Accordion>

  <Accordion title="Use correct platform identifiers">
    For Web, use the correct website URL. For Android, use the package name. For iOS, use the bundle ID.
  </Accordion>

  <Accordion title="Use production only for live data">
    If you are testing SDK setup, use a separate staging or testing project to avoid mixing test events with real users.
  </Accordion>

  <Accordion title="Check details before SDK integration">
    Make sure project details are correct before copying API keys or installing SDK.
  </Accordion>
</AccordionGroup>

---

## Common mistakes to avoid

<CardGroup cols={2}>
  <Card title="Wrong platform selected" icon="triangle-alert">
    Do not create a Web project for Android or an Android project for iOS.
  </Card>

  <Card title="Wrong domain or identifier" icon="circle-alert">
    Website URL, package name, and bundle ID should match your real product.
  </Card>

  <Card title="Using one project for all platforms" icon="combine">
    Avoid mixing Web, Android, and iOS data in one project unless your setup specifically requires it.
  </Card>

  <Card title="Using test project for production" icon="bug">
    Do not send live user data to a temporary testing project.
  </Card>
</CardGroup>

## What happens after project creation?

After your project is created, Twinalyze will generate project-level credentials and configuration details.

<Steps>
  <Step title="Open project settings">
    Go to the newly created project and open project settings.
  </Step>

  <Step title="Copy project credentials">
    Copy the required project ID, API key, or SDK key based on your integration.
  </Step>

  <Step title="Install SDK">
    Add the Web, Android, or iOS SDK to your application.
  </Step>

  <Step title="Send your first event">
    Trigger an event from your app or website and check if it appears in Twinalyze.
  </Step>
</Steps>
