# Create Project

A project is where Twinalyze collects analytics data for your app or website.

Create a separate project for each platform such as **Web**, **Android**, and **iOS**. This keeps your analytics clean and makes SDK setup easier.

<CardGroup cols={3}>
  <Card title="Web" icon="globe">
    Use this for websites, React apps, Shopify, WordPress, Webflow, and JavaScript SDK.
  </Card>

  <Card title="Android" icon="smartphone">
    Use this for Android apps with Kotlin or Java SDK integration.
  </Card>

  <Card title="iOS" icon="apple">
    Use this for iPhone or iPad apps with iOS SDK integration.
  </Card>
</CardGroup>

---

## Create project flow

<Steps>
  <Step title="Open project dropdown">
    Click the project selector from the left sidebar.
  </Step>
  <Step title="Click Create Project">
    Select the **Create Project** option from the dropdown.
  </Step>
  <Step title="Enter project name">
    Add your project name, for example `Twinsho Web`, `Twinsho Android`, or `Twinsho iOS`.
  </Step>
  <Step title="Save project">
    Click **Save** to create your project.
  </Step>
  <Step title="Continue setup">
    After creating the project, configure platform details and start SDK integration.
  </Step>
</Steps>

## Step 1: Open project dropdown

Click the project dropdown from the left sidebar.\
Here you can see existing projects and the **Create Project** option.

## Step 2: Create project

Click **Create Project**, enter your project name, and save it.

<Frame>
  ![Create project form in Twinalyze](/images/create-project-step-1.png)
</Frame>

<Info>
  Use a clear project name so your team can easily identify the platform later.
</Info>

Good project name examples:

| Platform | Recommended Project Name |
| --- | --- |
| Web | `Twinsho Web` |
| Android | `Twinsho Android` |
| iOS | `Twinsho iOS` |

## Choose project type

<Tabs>
  <Tab title="Web">
    ## Web Project

    Create a **Web** project when you want to track a website, web app, React app, Shopify store, WordPress website, or Webflow site.

    <CardGroup cols={2}>
      <Card title="Required details" icon="list-checks">
        Project name, website URL, platform type, and timezone.
      </Card>

      <Card title="Next setup" icon="code">
        Install JavaScript SDK, React SDK, or connect your website platform.
      </Card>
    </CardGroup>

    Example:

    ```json
    {
      "projectName": "Twinsho Web",
      "platform": "Web",
      "websiteUrl": "https://twinsho.com"
    }
    ```

    <Card title="Continue with Web SDK" icon="globe" href="/sdks/web/javascript/analytics">
      Start Web SDK integration after creating your Web project.
    </Card>
  </Tab>
  <Tab title="Android">
    ## Android Project

    Create an **Android** project when you want to track an Android app using Kotlin or Java SDK.

    <CardGroup cols={2}>
      <Card title="Required details" icon="list-checks">
        Project name, app name, package name, platform type, and timezone.
      </Card>

      <Card title="Next setup" icon="smartphone">
        Install Android SDK and start tracking screens, sessions, events, and ads.
      </Card>
    </CardGroup>

    Example:

    ```json
    {
      "projectName": "Twinsho Android",
      "platform": "Android",
      "appName": "Twinsho",
      "packageName": "com.twinsho.app"
    }
    ```

    <Warning>
      Make sure your package name is correct. It should match your real Android app package name.
    </Warning>

    <Card title="Continue with Android SDK" icon="smartphone" href="/sdks/android/kotlin/analytics">
      Start Android SDK integration after creating your Android project.
    </Card>
  </Tab>
  <Tab title="iOS">
    ## iOS Project

    Create an **iOS** project when you want to track an iPhone or iPad app.

    <CardGroup cols={2}>
      <Card title="Required details" icon="list-checks">
        Project name, app name, bundle ID, platform type, and timezone.
      </Card>

      <Card title="Next setup" icon="apple">
        Install iOS SDK and start tracking users, sessions, events, and ads.
      </Card>
    </CardGroup>

    Example:

    ```json
    {
      "projectName": "Twinsho iOS",
      "platform": "iOS",
      "appName": "Twinsho",
      "bundleId": "com.twinsho.ios"
    }
    ```

    <Warning>
      Make sure your Bundle ID is correct. It should match your real iOS app Bundle Identifier.
    </Warning>

    <Card title="Continue with iOS SDK" icon="apple" href="/sdks/ios/analytics">
      Start iOS SDK integration after creating your iOS project.
    </Card>
  </Tab>
</Tabs>

## Best practices

<AccordionGroup>
  <Accordion title="Create separate projects for each platform">
    Keep Web, Android, and iOS projects separate. This makes reports cleaner and SDK management easier.
  </Accordion>

  <Accordion title="Use clear project names">
    Use names like `Brand Web`, `Brand Android`, and `Brand iOS` instead of generic names like `Test` or `Project 1`.
  </Accordion>

  <Accordion title="Use correct platform details">
    For Web, add the correct website URL. For Android, use the correct package name. For iOS, use the correct Bundle ID.
  </Accordion>
</AccordionGroup>
