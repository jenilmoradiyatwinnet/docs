Twinalyze is an analytics and engagement platform that helps you understand how users interact with your app or website.

It collects product activity like users, events, sessions, screens, sources, campaigns, notifications, and device data, then turns it into clear reports for product, marketing, and development teams.

## Why Twinalyze is useful

Twinalyze helps teams answer important questions without checking multiple tools.

| Question | Twinalyze helps you see |
| --- | --- |
| Who is using my product? | Users, devices, country, language, app version |
| What are users doing? | Events, clicks, screens, pages, sessions |
| Where do users come from? | Sources, campaigns, referrals, UTM data |
| Are users staying active? | Retention, engagement, session activity |
| Are campaigns working? | Notifications, conversions, campaign reports |
| Is tracking correct? | Missing events, duplicate events, invalid properties |

## What Twinalyze tracks

Twinalyze can track both automatic and custom activity.

<CardGroup cols={2}>
  <Card title="Automatic tracking" icon="wand-magic-sparkles">

  </Card>

  <Card title="Custom events" icon="cursor">
    Track business actions like signup, login, add to cart, purchase, payment failed, plan upgraded, or feature used.
  </Card>
</CardGroup>

## Main data types

| Data type | Meaning | Example |
| --- | --- | --- |
| **User** | Person using your product | `user_1024`, email, name |
| **Event** | Action done by user | `add_to_cart`, `purchase_completed` |
| **Session** | One usage period | User opened app and used it for 5 minutes |
| **Screen / Page** | Place user visited | Home page, product screen, checkout page |
| **Property** | Extra detail | price, category, country, plan |
| **Source** | Where user came from | Google Ads, organic, referral, UTM |
