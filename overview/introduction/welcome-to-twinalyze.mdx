This documentation helps you learn how to set up Twinalyze, understand your product data, explore analytics reports, manage engagement tools, and integrate SDKs or APIs into your app or website.

### What you will find here

The docs are organized into sections based on how teams use Twinalyze.

| Section | What it covers |
| --- | --- |
| **Overview** | Basic concepts, use cases, and getting started |
| **Analytics** | Dashboards, realtime users, reports, filters, and metrics |
| **Engagement** | Notifications, campaigns, ad analytics, research, and Ask AI |
| **Data** | Events, properties, sources, controls, and data quality |
| **Developers** | SDKs, tracking, integrations, APIs, and troubleshooting |
| **Admin** | Project settings, billing, security, privacy, and white-label setup |

## Recommended path

If you are new to Twinalyze, start with this order:

1. Read the core concepts.
2. Create your account.
3. Create your organization and project.
4. Get your API key.
5. Install an SDK or connect an integration.
6. Send your first event.
7. Verify your data in the dashboard.
