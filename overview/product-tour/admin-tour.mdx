Admin tools help you control how Twinalyze is configured and managed across your organization.

> Use this section to manage **settings, access, billing, security, privacy, and branding**.

## What admins can manage

<CardGroup cols={3}>
  <Card title="Project settings" icon="gear">
    Manage project overview, API keys, Firebase keys, integrations, and versions.
  </Card>

  <Card title="Team access" icon="users">
    Invite members, assign roles, and control organization or project access.
  </Card>

  <Card title="Security" icon="shield">
    Manage API security, privacy, data retention, PII handling, and compliance.
  </Card>
</CardGroup>

## Admin workflow

<Steps>
  <Step title="Review project settings">
    Check project identity, API keys, Firebase SDK keys, and integration settings.
  </Step>

  <Step title="Invite team members">
    Add developers, analysts, marketers, or stakeholders to your organization.
  </Step>

  <Step title="Assign roles">
    Give each member the correct access level based on their responsibility.
  </Step>

  <Step title="Configure billing">
    Review plans, credits, payments, and refund or cancellation settings.
  </Step>

  <Step title="Secure the workspace">
    Review privacy, API security, data retention, and white-label configuration.
  </Step>
</Steps>

## Main admin areas

<Tabs>
  <Tab title="Project Settings">
    Project Settings control how your project works.

    | Setting | Purpose |
    | --- | --- |
    | **Project Overview** | Basic project information |
    | **API Keys** | Keys used for SDK and API integration |
    | **Firebase SDK Keys** | Keys used for Firebase-related setup |
    | **Integration Settings** | Configure connected tools |
    | **Version Settings** | Manage app or SDK versions |
    | **Device Settings** | Control device-level tracking |
    | **Screen Activity Settings** | Manage screen tracking behavior |
  </Tab>

  <Tab title="Team & Access">
    Team & Access helps you manage collaboration.

    <CardGroup cols={2}>
      <Card title="Team members" icon="user-plus">
        Invite users and manage existing members.
      </Card>

      <Card title="Roles & permissions" icon="lock">
        Control who can view, edit, export, configure, or manage data.
      </Card>
    </CardGroup>

    <Tip>
      Give users only the access they need. This keeps analytics, billing, and settings safer.
    </Tip>
  </Tab>

  <Tab title="Billing">
    Billing helps manage plan usage and payments.

    <AccordionGroup>
      <Accordion title="Plans Overview">

      <Accordion title="Credit System">
        Understand how credits are used for events, usage, or add-ons.
      </Accordion>

      <Accordion title="Payment Issues">

      </Accordion>

      <Accordion title="Cancel & Refund Policy">
        Understand cancellation, refund, and account downgrade rules.
      </Accordion>
    </AccordionGroup>
  </Tab>

  <Tab title="Security & Privacy">

    <Check>Protect API and secret keys.</Check>

## Recommended role setup

| Role | Best for | Access level |
| --- | --- | --- |
| **Owner** | Founder / main account holder | Full access |
| **Admin** | Manager / operations lead | Settings and team management |
| **Developer** | Engineering team | SDK, API, events, integrations |
| **Analyst** | Product / marketing team | Reports and insights |
| **Viewer** | Stakeholders | Read-only access |

## Best practices

<Check>Keep billing and security access limited.</Check>
<Check>Use separate roles for developers, analysts, and viewers.</Check>
<Check>Review API keys and team access regularly.</Check>
<Check>Use white-label settings only for approved client domains.</Check>
<Check>Document internal ownership for each project.</Check>
