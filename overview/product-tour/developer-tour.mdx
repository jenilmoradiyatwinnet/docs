Developer tools help you connect your app, website, or backend with Twinalyze.

> Use this section to set up **SDKs, events, users, sessions, API keys, and debugging**.

## What developers can manage

<CardGroup cols={3}>
  <Card title="SDK setup" icon="code">
    Install and initialize Android, Web, or other SDKs.
  </Card>

  <Card title="Tracking" icon="bolt">
    Track auto events, custom events, users, sessions, and screens.
  </Card>

  <Card title="Authentication" icon="key">
    Use API keys, secret keys, request headers, and HMAC signatures.
  </Card>
</CardGroup>

## Developer workflow

<Steps>
  <Step title="Create project">
    Create a project and get your project keys from Project Settings.
  </Step>

  <Step title="Install SDK">
    Add the Twinalyze SDK to your app or website.
  </Step>

  <Step title="Initialize SDK">
    Initialize Twinalyze using your project API key.
  </Step>

  <Step title="Send first event">
    Track your first custom event and verify it in Realtime Users or Data.
  </Step>

  <Step title="Debug tracking">
    Use Debug Mode and troubleshooting guides to fix missing or duplicate events.
  </Step>
</Steps>

## Main developer areas

<Tabs>
  <Tab title="Quickstart">
    Quickstart helps you go from setup to first event quickly.

    | Page | Purpose |
    | --- | --- |
    | **Developer Overview** | Understand the developer setup flow |
    | **Install & Initialize SDK** | Add SDK and connect your project |
    | **Send First Event** | Verify your first event |
    | **Debug Mode** | Test and fix tracking issues |
  </Tab>

  <Tab title="SDKs">
    SDK pages explain platform-specific integration.

    <CardGroup cols={2}>
      <Card title="Android SDK" icon="mobile">
        Track app opens, screens, sessions, device details, and custom events.
      </Card>

      <Card title="Web SDK" icon="globe">
        Track page views, clicks, sessions, users, sources, and custom events.
      </Card>
    </CardGroup>
  </Tab>

  <Tab title="Tracking">
    Tracking pages explain how data is captured.

    <AccordionGroup>
      <Accordion title="Auto tracking">
        Automatically capture common activity like sessions, screens, app foreground, app background, and page views.
      </Accordion>

      <Accordion title="Manual tracking">
        Track important business actions like signup, purchase, add to cart, payment failed, or feature usage.
      </Accordion>

      <Accordion title="User identification">
        Connect anonymous activity with known users after login or signup.
      </Accordion>

      <Accordion title="Advanced screen tracking">
        Track activities, fragments, tabs, and complex navigation flows.
      </Accordion>
    </AccordionGroup>
  </Tab>

  <Tab title="Authentication">
    Authentication protects API and SDK communication.

    | Item | Meaning |
    | --- | --- |
    | **API Key** | Identifies the project |
    | **Secret Key** | Used for secure signing |
    | **Request Headers** | Pass platform, version, timestamp, and signature |
    | **HMAC Signature** | Verifies request authenticity |
    | **Rate Limits** | Protects system from abuse |
  </Tab>
</Tabs>

## Best practices

<Check>Use separate projects for different apps or environments.</Check>
<Check>Keep API keys and secret keys secure.</Check>
<Check>Use clear event names and consistent properties.</Check>
<Check>Verify tracking in Realtime Users after SDK setup.</Check>
<Check>Use Debug Mode before releasing tracking changes.</Check>

<Warning>
Never expose secret keys in frontend code or public repositories.
</Warning>
