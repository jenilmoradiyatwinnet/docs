# Settings

The **Settings** section helps users manage their personal profile, organization information, security preferences, login activity, users, and permissions from one central place.

This page is mainly used by organization members and administrators to keep account details updated, invite team members, and control access based on user roles.

## Manage Profile

The **Manage Profile** page allows users to update their basic account information and profile details.

Users can manage their name, email, mobile number, date of birth, address, country, state, city, and zip code. They can also upload or remove their profile picture.

<Note>
  Some fields such as email may be disabled because they are linked with the user account and cannot be changed directly.
</Note>

## Users & Permissions

The **Users & Permissions** page helps administrators manage team access inside the organization.

Admins can invite new users by email, assign a role type, refresh the user list, search users, and filter users by status or role.

This helps organizations keep access clean, secure, and role-based.

### What admins can do

<Steps>
  <Step title="Invite a new user">
    Enter the user's email address and select a role type to send an invitation.
  </Step>

  <Step title="Assign a role">
    Choose the correct role for the user based on their responsibility inside the organization.
  </Step>

  <Step title="Search and filter users">
    Use status, role, and search filters to quickly find existing users.
  </Step>

  <Step title="Review user details">
    Check user name, email, phone number, role type, and current status from the users table.
  </Step>
</Steps>

---

## Permissions

The **Permissions** tab is used to control what each role can access inside Twinalyze.

Permissions help define which users can view data, manage projects, invite users, update settings, access reports, or perform admin-level actions.

<Info>
  Role-based permissions help protect sensitive organization data and make sure each team member only has access to the features they need.
</Info>

## Best practices

<AccordionGroup>
  <Accordion title="Keep profile details updated">
    Update your name, mobile number, and address details whenever required so your account information stays accurate.
  </Accordion>

  <Accordion title="Use correct roles for users">
    Assign roles carefully based on each user's responsibility. Avoid giving admin access to users who only need view-level access.
  </Accordion>

  <Accordion title="Review users regularly">
    Check the users list regularly and remove or update access for users who no longer need access to the organization.
  </Accordion>

  <Accordion title="Monitor login activity">
    Use login history to review account access and identify any unusual login activity.
  </Accordion>
</AccordionGroup>
