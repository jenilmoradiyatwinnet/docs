## Manage Profile

The **Manage Profile** page allows users to update their personal and account-related information.

Users can manage their profile picture, first name, last name, mobile number, date of birth, address, country, state, city, and zip code.

<Note>
  Email may be disabled because it is connected with the registered account and is usually not editable from profile settings.
</Note>

## Permissions

The **Permissions** tab is used to control feature access for each role.

Permissions define what a user can view, create, edit, delete, invite, export, or manage inside Twinalyze.

<Info>
  Role-based permissions help protect organization data and make sure users only access the features required for their work.
</Info>

<AccordionGroup>
  <Accordion title="Why permissions are important">
    Permissions help prevent unauthorized access to sensitive organization data, reports, users, and project settings.
  </Accordion>

  <Accordion title="Who should manage permissions">
    Usually, owners and administrators should manage permissions because they control access for the full organization.
  </Accordion>

  <Accordion title="When to update permissions">
    Update permissions when a user's responsibility changes, when a new team member joins, or when a user no longer needs access.
  </Accordion>
</AccordionGroup>

## Best practices

<Steps>
  <Step title="Keep profile details updated">
    Make sure personal and address details are accurate so account information stays clean.
  </Step>

  <Step title="Use role-based access">
    Assign roles based on user responsibility instead of giving admin access to every user.
  </Step>

  <Step title="Review users regularly">
    Remove or update access for users who no longer need access to the organization.
  </Step>

  <Step title="Check login history">
    Review login history to monitor account access and identify unusual activity.
  </Step>
</Steps>
