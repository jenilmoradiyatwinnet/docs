System Insights helps you understand the technical environment of your users.

<Steps>
  <Step title="Review OS, brand, and RAM">
    The first section gives a quick technical summary of your users. You can see Android OS versions, device brands, and RAM distribution.

    This helps you understand which device environments are most common and where your testing should focus first.

  </Step>

  <Step title="Check app store and browser packages">
    This section shows the default app store package and default browser package used by users.

    It helps you understand where users install from and which browser environments matter most for redirects, deep links, webviews, and compatibility.

    This section shows device screen inch, screen density, and device type.

    It helps your team make better layout, UI testing, responsive design, and asset scaling decisions.

    It helps you understand connection quality, carrier distribution, theme usage, and localization opportunities.

## How to use System Insights

<Steps>
  <Step title="Select date and user type">
    Use the top filters to choose a date range and user type. This updates all system insight charts.
  </Step>

  <Step title="Start with device basics">
    Review Android OS, Brand, and RAM first to understand the most common technical environment.
  </Step>

  <Step title="Check store and browser packages">
    Review app store and browser package charts to understand install and browser compatibility context.
  </Step>

  <Step title="Review display and security signals">
    Check screen size, density, device type, developer option, proxy, and VPN status.
  </Step>

  <Step title="Analyze network and language">
    Review network, SIM name, screen mode, device language, app language, and unsupported languages.
  </Step>
</Steps>

## Common use cases

<AccordionGroup>
  <Accordion title="Plan testing coverage">
    Use Android OS, Brand, RAM, Density, and Device Screen Inch to decide which devices and environments need more testing.
  </Accordion>

  <Accordion title="Improve compatibility decisions">
    Use Default Browser Package, Default App Store Package, Device Type, and Screen Mode to improve app and web compatibility.
  </Accordion>

  <Accordion title="Find localization opportunities">
    Use Device vs App Language and Unsupportable App Language to identify where new language support may be needed.
  </Accordion>

  <Accordion title="Understand network conditions">
    Use Device Network and Device SIM Name to understand how users connect and which carriers are common.
  </Accordion>

  <Accordion title="Investigate unusual environments">
    Use Developer Option, Proxy, and VPN charts to identify special technical setups or QA-like users.
  </Accordion>
</AccordionGroup>

## Quick reading tips

<Tip>
  Start with **Android OS, Brand, and RAM** for the fastest understanding of your user device base.
</Tip>

<Tip>
  If one browser, brand, OS version, or screen size dominates, prioritize testing and optimization there first.
</Tip>

<Tip>
  If device language and app language do not match, review language detection, onboarding, and localization support.
</Tip>

<Check>
  System Insights gives your team a technical view of your users across devices, operating systems, packages, network, security signals, screen behavior, and language usage.
