# Advanced Report

Advanced Report helps you build custom reports from your tracked events.

Use it when you want to compare event trends, filter users, group data by properties, measure funnel conversion, and save the report for later use.

<Note>
  Advanced Report currently has two main report modes: **Segmentation** and **Funnel**.
</Note>

## Choose the right report mode

<Tabs>
  <Tab title="Use Segmentation when">
    Use **Segmentation** when you want to understand how events perform over time.

    Best for:

    - comparing one event with another
    - checking event growth or drop
    - filtering users by properties
    - grouping results by country, city, device, OS, brand, or other properties
    - reviewing exact values in the Overview table

    <Tip>
      Use Segmentation when your question starts with: **“How many users did this event?”** or **“How did this event change over time?”**
    </Tip>
  </Tab>

  <Tab title="Use Funnel when">
    Use **Funnel** when you want to understand how users move through steps.

    Best for:

    - checking conversion between events
    - finding where users drop off
    - comparing step completion
    - measuring whether users completed the next action
    - reviewing conversion percentage in the Funnel Table

    <Tip>
      Use Funnel when your question starts with: **“How many users moved from step A to step B?”**
    </Tip>
  </Tab>
</Tabs>

## Segmentation

Segmentation helps you compare event activity over time.

Use it when you want to understand whether an event increased, dropped, or changed across selected dates.

### Segmentation screen areas

<Steps>
  <Step title="Add one or more events">
    Select an event from the event list. Each selected event appears with a label such as **A**, **B**, or **C**.

    This makes it easy to compare multiple events in the same chart.
  </Step>

  <Step title="Use Filter by">
    Use **Filter by** to narrow the event data.

    Example: show only users from a specific country, city, Android OS, brand, or device property.
  </Step>

  <Step title="Use Group by">
    Use **Group by** to split one event into multiple segments.

    Example: group **Fragment Viewed** by country to compare which countries generated more activity.
  </Step>

  <Step title="Choose chart type">
    Select the chart view based on the type of analysis you need.

    - **Line Chart** for trends
    - **Bar Chart** for comparison
    - **Stacked Bar** for grouped totals
    - **Pie Chart** for share-based view
  </Step>

  <Step title="Review Overview table">
    The Overview table shows exact event values by date.

    Use it when you need precise numbers, search, export, or row-level comparison.
  </Step>
</Steps>

## Funnel

Funnel helps you measure conversion from one event step to another.

Use it when you want to understand how many users completed the next action after the first action.

### Funnel screen areas

<Steps>
  <Step title="Add funnel steps">
    Add events one by one. Each event becomes a step in the funnel.

    Example:

    - Step A: Fragment Viewed
    - Step B: Back Screen
  </Step>

  <Step title="Choose order logic">
    Use order logic such as **Any order** to control how user movement should be counted.
  </Step>

  <Step title="Apply filters">
    Add filters to a funnel step when you want to measure conversion for a specific audience or condition.
  </Step>

  <Step title="Review conversion chart">
    The chart shows conversion from the first step.

    The first event starts at **100%**. The next step shows how many users continued from the first step.
  </Step>

  <Step title="Review Funnel Table">
    The Funnel Table shows conversion percentage and exact user counts for each step.
  </Step>
</Steps>

## Save report

Save keeps the current report configuration available for later.

Use Save when the same report will be used repeatedly by your team.

<Steps>
  <Step title="Build the report">
    Select Segmentation or Funnel, add events, filters, groups, date range, and chart settings.
  </Step>

  <Step title="Check the output">
    Review the chart and table to confirm the report is correct.
  </Step>

  <Step title="Click Save">
    Save the report configuration for future access.
  </Step>
</Steps>

<Tip>
  Save the report after the setup is final. This avoids rebuilding the same event, filter, and chart configuration again.
</Tip>

## Quick guide

  <Accordion title="When should I use Segmentation?">
    Use Segmentation when you want to compare event trends, event counts, or grouped event breakdowns over time.

    Use Funnel when you want to measure how users move from one event step to another.
