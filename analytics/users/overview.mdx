User Overview helps you inspect individual users, review their activity, check device details, and understand how they move through your app or website.

<Steps>
  <Step title="Open Users list">
    The Users page shows all tracked users with country, retention status, app status, app version, brand, last activity, and registration date.

    <Frame>
      <img
        src="/images/User_overview_1.PNG"
        alt="Twinalyze Users list page"
      />
    </Frame>
  </Step>

  <Step title="Open user overview">
    Click any user row to open the detailed user overview page. This view shows session summary, device properties, event properties, traffic source, and screen journey.

    <Frame>
      <img
        src="/images/Users_overview_2.PNG"
        alt="Twinalyze User Overview page"
      />
    </Frame>
  </Step>

  <Step title="Expand activity details">
    Expand a session row to view detailed activity such as screen views, fragment views, back screens, splash time, total time, and event properties.

    <Frame>
      <img
        src="/images/Users_overview_3.PNG"
        alt="Twinalyze User activity details"
      />
    </Frame>
  </Step>
</Steps>

## User flow

<Steps>
  <Step title="Open Users">
    Go to **Users** from the left sidebar to view the tracked user list.
  </Step>

  <Step title="Filter the list">
    Use date range, country filter, and search to find the right user.
  </Step>

  <Step title="Open user details">
    Click a user row to open the user overview page.
  </Step>

  <Step title="Review activity">
    Check session summary, total screens, total time, and date-wise activity.
  </Step>

  <Step title="Inspect properties">
    Review device properties, event properties, traffic source, and journey path.
  </Step>
</Steps>

## Common use cases

<AccordionGroup>
  <Accordion title="Debug a user journey" icon="route">
    Open a user and review journey path, event properties, and session activity to understand what happened.
  </Accordion>

  <Accordion title="Check device information" icon="smartphone">
    Use Device Properties to review app version, brand, model, language, IP address, screen size, and device ID.
  </Accordion>
