Messaging helps you create targeted push notifications for your app users.

<Steps>
  <Step title="Open Messaging">
    Open the **Messaging** section from the sidebar.

    The page may show a loading state while notification templates, filters, language data, and campaign setup options are being prepared.

    <Frame>
      <img
        src="/images/Messaging_overview_1.png"
        alt="Twinalyze Messaging loading state"
      />
    </Frame>
  </Step>

  <Step title="Select an event to continue">
    Use the left-side **Filters** panel to select the audience.

    First choose the date range, then click **Add Events**. The selected event controls which user group will be targeted for the notification.

    Until an event is selected, the editor shows an empty state and asks you to select an event to continue.

    <Frame>
      <img
        src="/images/Messaging_overview_2.png"
        alt="Twinalyze Messaging select event empty state"
      />
    </Frame>
  </Step>

  <Step title="Create the default notification">
    After selecting an event, the notification editor becomes active.

    Write the default notification content, including title, description, logo, vibrance, and launch URL. The phone preview updates so you can check how the notification will appear to users.

    <Frame>
      <img
        src="/images/Messaging_overview_3.png"
        alt="Twinalyze Messaging default notification editor"
      />
    </Frame>
  </Step>

  <Step title="Review language-wise content">
    Use the language panel to review translated notification content for each language.

    The green check indicates translated content. The yellow dot indicates edited content. This helps you quickly identify which languages are ready and which ones were manually changed.

    <Frame>
      <img
        src="/images/Messaging_overview_4.png"
        alt="Twinalyze Messaging language-wise notification editor"
      />
    </Frame>
  </Step>
</Steps>

## Launch URL options

Launch URL controls what happens when a user taps the notification.

<div className="not-prose overflow-hidden rounded-3xl border border-slate-200 bg-white shadow-sm">
  <div className="grid gap-0 md:grid-cols-3">
    <div className="border-b border-slate-100 p-6 md:border-b-0 md:border-r">
      <div className="text-sm font-semibold text-blue-700">Default App</div>
      <p className="mt-2 text-sm leading-6 text-slate-600">
        Opens the app normally when the user taps the notification.
      </p>
    </div>

    <div className="border-b border-slate-100 p-6 md:border-b-0 md:border-r">
      <div className="text-sm font-semibold text-blue-700">Click URL</div>
      <p className="mt-2 text-sm leading-6 text-slate-600">
        Opens a specific URL when the user taps the notification.
      </p>
    </div>

    <div className="p-6">
      <div className="text-sm font-semibold text-blue-700">Custom Screen</div>
      <p className="mt-2 text-sm leading-6 text-slate-600">
        Opens a selected screen inside the app for a more direct user journey.
      </p>
    </div>
  </div>

## Common use cases

<AccordionGroup>
  <Accordion title="Send offers and promotions">
    Create campaign messages for discounts, new arrivals, seasonal offers, or limited-time deals.
  </Accordion>

  <Accordion title="Re-engage inactive users">
    Target users who performed an event earlier but did not return recently.
  </Accordion>

  <Accordion title="Send language-wise notifications">
    Use translations to send the same campaign in each user’s preferred language.
  </Accordion>

  <Accordion title="Drive users to a specific screen">
    Use Custom Screen launch URL to open a specific app screen after notification click.
  </Accordion>

  <Accordion title="Test before sending">
    Use Test Notification to verify content, image, layout, and click behavior before scheduling.
  </Accordion>
</AccordionGroup>
