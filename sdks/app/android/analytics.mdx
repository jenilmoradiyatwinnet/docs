# Android Analytics SDK

Our SDK helps you understand **how users interact with your app**, track screens, live events, sessions, session durations, app states, and more with almost zero setup.

Whether you're building an Android app, our SDK seamlessly integrates with your workflow, delivering powerful results with minimal overhead.

---

## What’s Inside?

We offer **two SDK options** tailored to different needs:

|  |  |
| --- | --- |
| <strong>Analytics SDK</strong> | Tracks screens, user behavior, fragments, tabs, app in background/foreground, and sends notifications and more. |
| <strong>Ad Analytics SDK</strong> | Track live ad events, including requests, impressions, failures, revenue, and more, all in real time. Our SDK also supports individual user-level ad tracking, helping you identify and reduce ad failures for specific users and ensure better performance across your app. |

> Both SDKs share a common foundation, initialize once and start tracking.

---

## What You'll Learn

- How to integrate the SDK in minutes
- What events are tracked automatically
- How to manually track custom behavior and events
- How to use our Ad Analytics SDK if you monetize via ads

## SDK Initialization

### Add the SDK Dependency

Add the following dependency to your app-level `build.gradle` file.

<CodeGroup>

```kotlin Kotlin
implementation("com.twinalyze:analytics:1+")
```

```java Java
implementation("com.twinalyze:analytics:1+");
```

</CodeGroup>

In your **project-level** `build.gradle` make sure the following plugins are configured:

<CodeGroup>

```kotlin Kotlin
plugins {
    id("com.android.application") version "8.13.2" apply false
    id("org.jetbrains.kotlin.android") version "2.0.21" apply false
    id("org.jetbrains.kotlin.plugin.compose") version "2.0.21" apply false
}
```

```java Java
plugins {
    id 'com.android.application' version '8.13.2' apply false
    id 'org.jetbrains.kotlin.android' version '2.0.21' apply false
    id 'org.jetbrains.kotlin.plugin.compose' version '2.0.21' apply false
}
```

</CodeGroup>

### Initialize Analytics SDK in Android

Initialize the **SDK** in your `Application` class to start tracking sessions, events, and insights.

<CodeGroup>


```java Java
        @Override
```

## Auto and Manual Event Tracking

Our SDK supports automatic tracking for key user behavior events out of the box. In cases where automatic tracking may not be supported on certain devices or custom flows, manual tracking can be used.

### Events Automatically Tracked

These events are tracked by default, with no code required.

|  |  |
| --- | --- |
| <strong>Screen Viewed</strong> |
| <strong>Fragment Viewed</strong> | Logs views for individual fragments within an activity, such as . |
| <strong>Tab View</strong> | Tracks user switching between bottom or custom tab components, such as Home, Cart, or Profile. |
| <strong>Drawer View</strong> | Tracks drawer open and close events, useful for identifying navigation engagement. |
| <strong>Dialog</strong> | Tracks dialog open and close events, such as alerts, pop-ups, or confirmation dialogs. |
| <strong>App in Foreground</strong> | Fires when the app enters the foreground, such as when the user opens or resumes the app. |
| <strong>App Language</strong> | Tracks the app’s current language on first launch or when changed by the user. Useful for understanding localization trends. |
| <strong>Splash Time</strong> | Captures time spent on the splash screen to help analyze loading performance. |

---

## Custom Event Tracking

Use custom event tracking to capture user actions that matter to your app, such as button taps, page views, purchases, or any other interaction.

---

## How to Send a Custom Event

### Send a Single Field

<CodeGroup>

```kotlin Kotlin
Twinalyze.setCustomEvent("addtocart")
```

```java Java
Twinalyze.setCustomEvent("addtocart");
```

</CodeGroup>

### Send an Object

<CodeGroup>

```kotlin Kotlin
val eventProperties = JSONObject().apply {
    put("productId", "125")
    put("productName", "Headphones")
    put("currency", "USD")
    put("amount", 249.99)
}

Twinalyze.setCustomEvent("addtocart", eventProperties)
```

```java Java
JSONObject eventProperties = new JSONObject();

eventProperties.put("productId", "125");
eventProperties.put("productName", "Headphones");
eventProperties.put("currency", "USD");
eventProperties.put("amount", 249.99);

Twinalyze.setCustomEvent("addtocart", eventProperties);
```

</CodeGroup>

<Accordion title="Supported Data Types for Event Properties">
  |  |  |
  | --- | --- |
  |  |  |
  | / |  |
  |  |  |
  | / |  |
  |  |  |
  |  |  |
  |  | Not allowed. Avoid using null in . Use "0", "unknown", or remove the key. |
</Accordion>

> 📝 **Note**\
> Only primitive types, `JSONArray`, and `JSONObject` are allowed inside `eventProperties`. Avoid passing complex objects or custom class instances directly. Convert them to JSON first. All properties must be JSON-serializable. Also, avoid passing `null` values.

---

## Example of Custom Event Payload: Supported Data Types

Understand the types of data you can pass inside `eventProperties` and how to structure complex objects.

<CodeGroup>

```kotlin Kotlin
val eventProps = JSONObject().apply {
    put("event", "add_to_cart")
    put("productId", "125")
    put("productName", "Bluetooth Speaker X10")
    put("price", 129.99)
    put("currency", "USD")
    put("quantity", 2)
    put("category", "Audio Equipment")
    put("isInStock", true)

    val tags = JSONArray().apply {
        put("bluetooth")
        put("portable")
        put("bass")
    }
    put("tags", tags)

    val userInfo = JSONObject().apply {
        put("userId", "user_456789")
        put("membershipLevel", "Platinum")
    }
    put("userInfo", userInfo)
}

Twinalyze.setCustomEvent("addtocart", eventProps)
```

```java Java
JSONObject eventProps = new JSONObject();

eventProps.put("event", "add_to_cart");
eventProps.put("productId", "125");
eventProps.put("productName", "Bluetooth Speaker X10");
eventProps.put("price", 129.99);
eventProps.put("currency", "USD");
eventProps.put("quantity", 2);
eventProps.put("category", "Audio Equipment");
eventProps.put("isInStock", true);

JSONArray tags = new JSONArray();
tags.put("bluetooth");
tags.put("portable");
tags.put("bass");
eventProps.put("tags", tags);

JSONObject userInfo = new JSONObject();
userInfo.put("userId", "user_456789");
userInfo.put("membershipLevel", "Platinum");
eventProps.put("userInfo", userInfo);

Twinalyze.setCustomEvent("addtocart", eventProps);
```

</CodeGroup>

<Accordion title="Best Practices for Custom Event Tracking">
  Follow these guidelines to help users name, structure, and send custom events effectively.

  1. Naming Guidelines
     - Use `camelCase` or `snake_case` for event names, such as `purchaseCompleted` or `user_logged_in`
     - Keep names **short and descriptive**
     - Avoid **spaces** or special characters
  2. Event Structure
     - Use consistent keys, such as always using `"productId"`
     - Include only **relevant attributes**
     - Skip redundant or unused properties
  3. Avoid These Mistakes
     - Don’t send raw classes — always serialize to JSON
     - Avoid high-cardinality keys, such as UUIDs or timestamps
     - Never pass `null` — use `"unknown"` or remove the key
</Accordion>
